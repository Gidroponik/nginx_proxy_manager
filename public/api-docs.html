API Documentation

REST API для управления Nginx

REST JSON

Введение

API для программного управления конфигурациями Nginx. Позволяет добавлять, удалять и просматривать домены, управлять SSL сертификатами через простые HTTP запросы.

Base URL
https://your-server.com/api/v1
Безопасность

Все запросы требуют API ключ

Формат

JSON для запросов и ответов

Авторизация

Для всех запросов необходимо передавать API ключ в заголовке X-API-Key.

Пример заголовка
X-API-Key: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx-xxxxxxxx
Безопасность ключей

API ключи создаются в панели управления. Храните ключи в безопасности.

Привязка к IP

API ключи могут быть привязаны к конкретному IPv4 интерфейсу сервера.

Коды ошибок

MISSING_API_KEY 401

API ключ не передан

INVALID_API_KEY 401

Неверный API ключ

ACCESS_DENIED 403

Нет доступа к домену

DOMAIN_EXISTS 409

Домен уже существует

DOMAIN_NOT_FOUND 404

Домен не найден

INTERNAL_ERROR 500

Внутренняя ошибка сервера

Код HTTP Описание
MISSING_API_KEY 401 API ключ не передан
INVALID_API_KEY 401 Неверный API ключ
ACCESS_DENIED 403 Нет доступа к домену (другой IP)
MISSING_REQUIRED_FIELDS 400 Не переданы обязательные поля
DOMAIN_EXISTS 409 Домен уже существует
DOMAIN_NOT_FOUND 404 Домен не найден
PROTECTED_CONFIG 400 Защищённый конфиг (default)
MISSING_SSL_DATA 400 Не переданы данные SSL
INTERNAL_ERROR 500 Внутренняя ошибка сервера
Endpoints
GET /api/v1/interfaces

Возвращает список доступных IPv4 интерфейсов сервера для привязки доменов.

Пример запроса 

curl -X GET https://your-server.com/api/v1/interfaces \
  -H "X-API-Key: your-api-key"

Ответ (полный доступ) 

{
  "success": true,
  "data": [
    { "interface": "eth0", "address": "192.168.1.100" }
  ],
  "bound_to": null
}

Ответ (привязка к IP) 

{
  "success": true,
  "data": [
    { "interface": "eth0", "address": "192.168.1.100" }
  ],
  "bound_to": "192.168.1.100"
}
POST /api/v1/domain/add

Создаёт новую конфигурацию nginx для домена с настройкой SSL.

Параметры запроса

domain ● обяз.

Доменное имя (string)

proxy_pass ● обяз.

URL бэкенда (string)

websocket_path ○ опц.

Путь для WebSocket (string)

ssl_type ○ опц.

letsencrypt | selfsigned | custom

ssl_cert / ssl_key * custom

Сертификат и ключ для custom SSL

Параметр Тип Обяз. Описание
domain string ● Доменное имя
proxy_pass string ● URL бэкенда
websocket_path string ○ Путь для WebSocket
ssl_type string ○ letsencrypt | selfsigned | custom
ssl_cert string * Сертификат (для custom)
ssl_key string * Приватный ключ (для custom)

Пример запроса 

curl -X POST https://your-server.com/api/v1/domain/add \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{
    "domain": "example.com",
    "proxy_pass": "http://127.0.0.1:3000",
    "ssl_type": "letsencrypt"
  }'

Успешный ответ 

{
  "success": true,
  "data": {
    "domain": "example.com",
    "filename": "example.com.conf",
    "ssl_status": "ok",
    "ssl_type": "letsencrypt"
  }
}
POST /api/v1/domain/delete

Удаляет конфигурацию домена и перезагружает nginx.

Параметры запроса

domain string — Доменное имя или имя файла конфига

Пример запроса 

curl -X POST https://your-server.com/api/v1/domain/delete \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{ "domain": "example.com" }'

Успешный ответ 

{
  "success": true,
  "data": { "domain": "example.com", "deleted": true }
}
GET /api/v1/domain/list

Возвращает список доменов с информацией о SSL.

Пример запроса 

curl -X GET https://your-server.com/api/v1/domain/list \
  -H "X-API-Key: your-api-key"

Пример ответа 

{
  "success": true,
  "data": [
    {
      "domain": "example.com",
      "filename": "example.com.conf",
      "proxy_pass": "http://127.0.0.1:3000",
      "ssl_status": "ok",
      "ssl_type": "letsencrypt",
      "listen_ip": "192.168.1.100"
    }
  ],
  "count": 1,
  "bound_to": null
}

Поля ответа

domain Доменное имя
proxy_pass URL бэкенда
ssl_status ok | error | none
listen_ip IP привязки | null
bound_to IP привязки ключа
Поле Тип Описание
domain string Доменное имя
filename string Имя конфиг-файла
proxy_pass string URL бэкенда
ssl_status string ok | error | none
ssl_type string letsencrypt | custom | none
listen_ip string|null IP привязки домена
bound_to string|null IP привязки API ключа
